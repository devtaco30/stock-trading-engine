# 📱 NPL 공동투자 플랫폼 - 프로젝트 시나리오

## 1. 서비스 개요

### 1-1. 플랫폼 컨셉
- **이름**: (미정)
- **컨셉**: 개인 투자자들이 NPL에 소액으로 공동 투자하여 수익을 추구하는 플랫폼
- **차별점**: 기관만 참여하던 NPL 시장에 개인들이 소액으로 진입

### 1-2. 플랫폼 역할
- NPL 상품 발굴 및 등록
- 투자자 모집 (앱)
- 투자금 관리 및 실행
- 수익 정산 및 배당

### 1-3. 수익 구조

**투자자 입장:**
```
1. 앱에서 투자 상품 선택
2. 투자금 입금 (예: 1,000만 원)
3. 상품 확정 후 지분 보유
4. 배당 받기 (6개월~1년 후)
```

**플랫폼 입장:**
```
1. NPL 상품 등록
2. 투자자 모집 (앱)
3. 수수료 수입:
   - 운용 수수료: 투자금의 2%
   - 성과보수: 수익의 20%
```

## 2. 핵심 제약사항

### 2-1. 사모 투자 모델 (Phase 1)

**49명 제한 이유:**
- 자본시장법: 50인 이상 시 공모로 간주, 증권신고서 제출 필요
- 초기 단계에서는 규제 부담을 줄이기 위해 49명 제한
- 이로 인해 **선착순 경쟁이 핵심 이슈**

**최소 투자금:**
- 1인당 최소: 100만 원 이상
- 상품별 총액: 1억 원 이상

### 2-2. 공모 투자 모델 (Phase 2)

**특징:**
- 인원 제한 없음 (증권신고서 제출 후)
- 소액 투자 가능 (1만 원 단위)
- 1인당 연간 투자 한도: 2,000만 원

> **구현 참고:** Phase 1에서는 사모 모델(49명 제한)만 구현하고, Phase 2에서 공모 모델 확장 예정

## 3. 유저 시나리오: 사모 모델 (49인 제한) ⭐️ Phase 1 메인

### 3-1. 플랫폼 상황 설정
- 회원 수: 100만 명
- 오픈 상품: 100개 (동시 오픈)
- 상품당 정원: 49명
- 오픈 시간: 매일 정오 12시

### 3-2. 상품 예시: "강남 아파트 NPL"

**물건 정보:**
- 위치: 서울 강남구 대치동
- 담보: 아파트 35평
- 채권 원금: 10억 원 (1순위 근저당)
- 채권 매입가: 7억 원 (70% 할인)
- 연체이자 누적: 1억 원
- 총 회수 예상액: 11억 원 (원금 10억 + 연체이자 1억)
- 감정가: 20억 원
- 목표 모집액: 7억 원
- 모집 정원: 49명
- 1인당 투자 금액: 1,429만 원 (균등 배분)
- 예상 수익률: 57% (11억 회수 시)
- 예상 기간: 6개월

**수익 시뮬레이션:**
```
[플랫폼 채권 매입]
- 채권 원금: 10억
- 연체이자 누적: 1억
- 플랫폼 매입가: 7억 (30% 할인)

[투자자 모집]
- 목표 모집액: 7억
- 49명 × 1,429만 원 = 7억

[정산 (경매 낙찰 가정)]
- 경매 낙찰가: 12억
- 채권자 배당: 11억 (원금 10억 + 연체이자 1억)
- 차액 1억은 채무자에게 반환

[수수료 및 배당]
- 회수액: 11억
- 플랫폼 수수료 (2%): 2,200만 원
- 투자자 배당: 10억 7,800만 원
- 투자자 수익: 10억 7,800만 - 7억 = 3억 7,800만
- 투자자 수익률: 54%
```

**Phase 5: 진행 상황 추적 (1~6개월)**

**Phase 6: 수익 정산 (6개월 후)**
```
[정산 내역]

[이현우 배당]
- 지분율: 2.04%
- 배당금: 10억 7,800만 × 2.04% = 2,199만 원
- 투자금: 1,429만 원
- 순수익: 770만 원
- 수익률: 54%
- 투자 기간: 6개월 → 연환산 108%
```

## 5. 핵심 요구사항 및 제약사항

### 5-1. 동시성 제어

**문제:**
- 10만 명 동시 접속
- 49명만 성공해야 함
- 정원 초과 절대 불가 (50명 이상 되면 안 됨)
- Race Condition 방지 필수

### 5-2. 트래픽 특성

**예상 트래픽:**
```
[피크 타임: 12:00:00 ~ 12:00:10]
- 동시 접속: 10만 명
- 초당 요청: 10,000 RPS
- 성공: 49명 (0.049%)
- 실패: 99,951명 (99.951%)
```

**요구사항:**
- 빠른 응답 (200ms 이내)
- 정확한 정원 관리
- 실패 유저에게 명확한 피드백

### 5-3. 투자 프로세스

**복잡한 상태 전이:**
```
1. 투자 신청 (선착순)
2. 30분 입금 대기
3. 입금 확인 → 확정
4. 미입금 → 자동 취소 → 대기자 승급
```

**요구사항:**
- 30분 타이머 정확도
- 자동 취소 처리
- 대기자 자동 알림

### 5-4. 대기자 관리

**요구사항:**
```
- 대기자 순번 관리 (선착순)
- 취소 발생 시 대기자에게 즉시 알림
- 재경쟁 기회 제공 (5분 제한)
- 순번 조회 기능
```

## 6. 구현 범위 정의 ⭐️

### 6-1. Phase 1: MVP (핵심 기능)

**구현할 기능:**
```
✅ 회원 인증
   - 회원가입/로그인
   - 간단한 본인 인증

✅ 투자 상품 관리
   - 상품 목록 조회
   - 상품 상세 조회
   - 상품 등록 (관리자)

✅ 투자 신청 (핵심)
   - 선착순 투자 신청
   - 동시성 제어
   - 정원 관리 (49명 정확히)
   - 입금 대기 상태 생성

✅ 입금 처리
   - 입금 확인 API
   - 투자 확정 처리
   - 미입금 자동 취소

✅ 대기자 관리
   - 대기자 등록
   - 순번 관리
   - 취소 발생 시 알림

✅ 투자 내역 조회
   - 내 투자 목록
   - 투자 상세 정보
   - 진행 상황 추적

✅ 정산 처리
   - 관리자가 정산 실행
   - 배당금 계산
   - 배당 지급
```

**구현 안 할 기능 (모킹/간소화):**
```
❌ 실제 NPL 채권 거래
   → 관리자가 상품 등록만

❌ 실제 경매 시스템
   → 관리자가 "정산" 버튼 클릭

❌ 실제 결제 연동
   → API로 "입금 완료" 처리

❌ 실시간 알림
   → 간단한 구현

❌ 법적 서류 관리
   → 텍스트 필드만

❌ 2차 시장 (유통)
   → Phase 2로 미룸

❌ 공모 모델
   → Phase 2로 미룸
```

## 8. 트래픽 시뮬레이션

### 8-1. 사모 모델의 트래픽

**상황:**
- 100개 상품 × 49명 정원 = 4,900명 성공
- 실제 시도자: 50만 명 동시 접근
- 초당 요청: 10,000 RPS (피크)
- 성공률: 0.98%
- 실패 처리: 495,100건

**유저 경험:**
- 대부분의 유저는 "마감" 메시지를 받음
- 인기 상품은 1초 내 마감
- 대기자 등록 경쟁도 치열
- 선착순 스트레스 높음

**백엔드 챌린지:**
- 대용량 트래픽 처리
- 동시성 제어
- 빠른 실패 응답 (Fail Fast)
- 우아한 실패 처리 (Graceful Degradation)
