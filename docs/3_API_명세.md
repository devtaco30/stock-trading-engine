# API 명세

## 인증

### 회원가입

**입력:**
- 이메일 (string, 필수)
- 비밀번호 (string, 필수, 8자 이상)
- 이름 (string, 필수)
- 전화번호 (string, 필수)

**출력:**
- 사용자 ID
- 계좌 ID (자동 생성된 계좌)
- 생성 일시

**처리:**
1. 이메일 중복 체크
2. 비밀번호 암호화 (BCrypt)
3. 사용자 정보 저장
4. 계좌 자동 생성 (초기 현금 0원)

**예외:**
- 이메일 중복: "이미 존재하는 이메일"
- 비밀번호 형식 오류: "비밀번호는 8자 이상"

---

### 로그인

**입력:**
- 이메일 (string, 필수)
- 비밀번호 (string, 필수)

**출력:**
- 액세스 토큰 (JWT)
- 리프레시 토큰
- 토큰 만료 시간
- 사용자 ID
- 계좌 ID

**처리:**
1. 이메일로 사용자 조회
2. 비밀번호 검증
3. JWT 토큰 생성
4. 리프레시 토큰 저장

**예외:**
- 사용자 없음: "존재하지 않는 계정"
- 비밀번호 불일치: "비밀번호가 틀렸습니다"

---

## 계좌

### 계좌 조회

**입력:**
- 계좌 ID (string, 필수)

**출력:**
- 계좌 ID
- 현금 잔액 (BigDecimal)
- 총 자산 평가액 (BigDecimal)
- 미결제 금액 (BigDecimal)
- 매도 예정금액 (BigDecimal)
- 계좌 상태 (ACTIVE, IN_ARREARS, FROZEN)
- 보유 주식 목록
  - 종목 코드
  - 종목명
  - 보유 수량
  - 평균 매수가
  - 현재가
  - 평가 금액
  - 손익률

**처리:**
1. 계좌 정보 조회
2. 보유 주식별 현재가 조회
3. 평가 금액 계산
4. 손익률 계산

**예외:**
- 계좌 없음: "존재하지 않는 계좌"

---

### 입금

**입력:**
- 계좌 ID (string, 필수)
- 입금액 (BigDecimal, 필수, 양수)

**출력:**
- 거래 ID
- 입금 후 잔액 (BigDecimal)
- 거래 일시

**처리:**
1. 입금액 검증 (양수)
2. 현금 잔액 증가
3. 입금 내역 기록

**예외:**
- 음수 금액: "입금액은 양수여야 합니다"

---

### 출금

**입력:**
- 계좌 ID (string, 필수)
- 출금액 (BigDecimal, 필수, 양수)

**출력:**
- 거래 ID
- 출금 후 잔액 (BigDecimal)
- 거래 일시

**처리:**
1. 출금 가능 금액 계산 (현금 - 미결제 금액)
2. 출금액 검증
3. 미수금 있으면 출금 불가
4. 현금 잔액 감소
5. 출금 내역 기록

**예외:**
- 잔액 부족: "출금 가능 금액 부족"
- 미수금 있음: "미수금이 있어 출금 불가"

---

### 매수 가능 금액 조회

**입력:**
- 계좌 ID (string, 필수)

**출력:**
- 현금 잔액 (BigDecimal)
- 보유 주식 대용가 (BigDecimal)
- 매도 예정금액 (BigDecimal)
- 총 매수력 (BigDecimal)
- 매수 가능 금액 (BigDecimal)

**처리:**
1. 현금 잔액 조회
2. 보유 주식별 현재가 조회
3. 대용가 계산 (현재가 x 수량 x 0.7)
4. 매도 예정금액 조회
5. 총 매수력 = 현금 + 대용가 + 매도 예정금액
6. 매수 가능 금액 = 총 매수력 / 0.4

**예외:**
- 계좌 없음: "존재하지 않는 계좌"

---

## 주문

### 매수 주문

**입력:**
- 계좌 ID (string, 필수)
- 종목 코드 (string, 필수)
- 주문 유형 (LIMIT, MARKET, 필수)
- 가격 (BigDecimal, 지정가인 경우 필수)
- 수량 (int, 필수, 양수)

**출력:**
- 주문 ID
- 주문 상태 (PENDING)
- 주문 일시
- 홀딩된 증거금 (BigDecimal)

**처리:**
1. 계좌 상태 체크 (동결 계좌는 주문 불가)
2. 종목 존재 여부 확인
3. 현재가 조회 (시장가인 경우)
4. 필요 금액 계산
5. 증거금 계산 (필요 금액 x 0.4)
6. 매수 가능 금액 체크
7. 증거금 홀딩 (현금 차감)
8. 주문 생성 (PENDING 상태)

**예외:**
- 계좌 동결: "거래 정지된 계좌"
- 종목 없음: "존재하지 않는 종목"
- 매수력 부족: "매수 가능 금액 부족"
- 음수 수량: "수량은 양수여야 합니다"

---

### 매도 주문

**입력:**
- 계좌 ID (string, 필수)
- 종목 코드 (string, 필수)
- 주문 유형 (LIMIT, MARKET, 필수)
- 가격 (BigDecimal, 지정가인 경우 필수)
- 수량 (int, 필수, 양수)

**출력:**
- 주문 ID
- 주문 상태 (PENDING)
- 주문 일시

**처리:**
1. 보유 주식 조회
2. 보유 수량 체크
3. 주문 생성 (PENDING 상태)

**예외:**
- 보유 주식 없음: "보유하지 않은 종목"
- 수량 부족: "보유 수량 부족"

---

### 주문 취소

**입력:**
- 주문 ID (string, 필수)

**출력:**
- 취소된 주문 ID
- 반환된 증거금 (BigDecimal, 매수 주문인 경우)

**처리:**
1. 주문 조회
2. 주문 상태 체크 (PENDING만 취소 가능)
3. 매수 주문인 경우 홀딩된 증거금 반환
4. 주문 상태를 CANCELLED로 변경

**예외:**
- 주문 없음: "존재하지 않는 주문"
- 이미 체결: "체결된 주문은 취소 불가"

---

### 주문 내역 조회

**입력:**
- 계좌 ID (string, 필수)
- 상태 (PENDING, FILLED, CANCELLED, 선택)
- 시작일 (date, 선택)
- 종료일 (date, 선택)

**출력:**
- 주문 목록
  - 주문 ID
  - 종목 코드
  - 종목명
  - 매수/매도 구분
  - 주문 유형
  - 가격
  - 수량
  - 상태
  - 주문 일시
  - 체결 일시 (체결된 경우)

**처리:**
1. 조건에 맞는 주문 조회
2. 최신순 정렬

---

## 체결 (내부 처리)

### 체결 처리

**입력:**
- 주문 ID
- 체결가 (BigDecimal)

**출력:**
- 체결 ID
- 체결 일시

**처리:**
1. 주문 조회
2. 주문 상태 체크 (PENDING만 체결 가능)
3. 매수인 경우:
   - 보유 주식에 추가 또는 수량 증가
   - 평균 매수가 재계산
   - 잔금 계산 (체결 금액 - 홀딩 증거금)
   - 미결제 레코드 생성 (T+2 결제일)
4. 매도인 경우:
   - 보유 주식 수량 감소
   - 매도 예정금액 생성 (T+2 입금일)
5. 주문 상태를 FILLED로 변경
6. 체결 내역 기록

---

## 결제

### 미결제 내역 조회

**입력:**
- 계좌 ID (string, 필수)

**출력:**
- 미결제 내역 목록
  - 결제 ID
  - 종목 코드
  - 결제 예정일 (T+2)
  - 결제 금액
  - 상태 (PENDING, SETTLED)

**처리:**
1. 계좌의 미결제 내역 조회
2. 결제 예정일순 정렬

---

### 결제 처리 (배치)

**입력:**
- 결제일 (date)

**출력:**
- 처리된 결제 건수
- 미수 발생 계좌 수

**처리:**
1. 해당 날짜가 결제일인 미결제 조회
2. 각 계좌별로:
   - 현금 잔액 확인
   - 매도 예정금액 입금 처리
   - 결제 금액 차감
   - 부족하면 미수금 처리
   - 충분하면 결제 완료
3. 결과 로깅

---

## 미수금

### 미수금 조회

**입력:**
- 계좌 ID (string, 필수)

**출력:**
- 미수금 ID
- 미수금 금액 (BigDecimal)
- 발생일
- 연체 일수
- 누적 이자 (BigDecimal)
- 상태 (IN_ARREARS, SETTLED)

**처리:**
1. 계좌의 미수금 조회
2. 연체 일수 계산 (현재일 - 발생일)

**예외:**
- 미수금 없음: null 반환

---

### 미수금 상환

**입력:**
- 미수금 ID (string, 필수)
- 상환 금액 (BigDecimal, 필수, 양수)

**출력:**
- 상환 후 잔여 미수금 (BigDecimal)
- 상환 일시

**처리:**
1. 미수금 조회
2. 현금 잔액 체크
3. 현금 차감
4. 미수금 감소
5. 미수금 0원이면:
   - 미수금 상태를 SETTLED로 변경
   - 계좌 상태를 ACTIVE로 변경
6. 상환 내역 기록

**예외:**
- 현금 부족: "현금 잔액 부족"
- 미수금 없음: "존재하지 않는 미수금"

---

### 미수금 발생 처리 (배치)

**입력:**
- 계좌 ID
- 부족 금액 (BigDecimal)

**출력:**
- 미수금 ID

**처리:**
1. 미수금 레코드 생성
2. 계좌 상태를 IN_ARREARS로 변경
3. 계좌 동결 (추가 매수 불가)

---

### 미수 이자 계산 (배치)

**입력:**
- 없음 (전체 미수 계좌 대상)

**출력:**
- 처리된 계좌 수

**처리:**
1. 미수금 있는 계좌 조회
2. 각 계좌별로:
   - 일 이자 계산 (미수금 x 0.0003)
   - 미수금에 누적

---

## 강제 청산

### 청산 대상 조회 (배치)

**입력:**
- 없음

**출력:**
- 청산 대상 계좌 목록
  - 계좌 ID
  - 미수금 금액
  - 연체 일수
  - 보유 주식 목록

**처리:**
1. IN_ARREARS 상태 계좌 조회
2. 연체 일수 >= 1인 계좌만
3. 보유 주식 있는 계좌만

---

### 청산 수량 계산 (내부)

**입력:**
- 미수금 금액 (BigDecimal)
- 종목 코드
- 보유 수량

**출력:**
- 청산 수량 (int)

**처리:**
1. 전일 종가 조회
2. 하한가 계산 (전일 종가 x 0.9)
3. 청산 수량 = 미수금 / 하한가 (올림)
4. 보유 수량 초과하면 보유 수량 전체

---

### 강제 매도 실행 (배치)

**입력:**
- 계좌 ID
- 종목 코드
- 청산 수량

**출력:**
- 강제 매도 주문 ID

**처리:**
1. 시장가 매도 주문 생성
2. 주문 유형: FORCED_SELL
3. 시초가 체결 대기
4. 체결 후:
   - 체결 금액으로 미수금 차감
   - 남은 금액은 현금으로 입금
   - 미수금 0원이면 계좌 정상화

---

## 시세

**입력:**

**출력:**
- 종목 코드

- 저가 (BigDecimal)
- 거래량 (long)

2. 없으면 외부 API 호출 후 캐싱

**예외:**
- 종목 없음: "존재하지 않는 종목"

---

### 종목 검색

**입력:**
- 검색어 (string, 필수)

**출력:**
- 종목 목록
  - 종목 코드
  - 종목명
  - 현재가

**처리:**
1. 종목명 또는 종목 코드로 검색
2. 최대 20개까지 반환

## 배치 작업

### 결제일 배치
- 실행 시간: 매일 03:00
- 처리: T+2 결제 도래 건 처리

### 강제 청산 배치
- 실행 시간: 매일 05:00
- 처리: 연체 계좌 강제 매도

### 이자 계산 배치
- 실행 시간: 매일 00:00
- 처리: 미수금 이자 계산

### 시세 업데이트
- 실행 주기: 실시간 또는 1분
- 처리: 현재가 갱신 및 캐싱
