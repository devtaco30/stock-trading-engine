# 주식 거래 엔진 - 4단계 시나리오

## 1단계: 기본 주문 및 체결

**시나리오:**
계좌 A가 삼성전자 100주 매수.

**처리 과정:**

### 주문 시점 (09:00)

**API 호출:**
- `GET /api/v1/accounts/{accountId}` - 계좌 조회 (매수 가능 금액 확인)
- `GET /api/v1/stocks/{stockCode}/quote` - 현재가 조회

**처리:**
- 삼성전자 현재가: 70,000원
- 지정가 주문: 70,500원에 100주 매수
- 필요 금액: 7,050,000원
- 필요 증거금 (40%): 7,050,000 × 0.4 = 2,820,000원

### 매수 가능 금액 체크
```
현금 잔액: 3,000,000원
증거금률 40% 기준:
- 7,500,000원어치 매수 시 증거금 필요: 7,500,000 × 0.4 = 3,000,000원
- 따라서 매수 가능 금액: 7,500,000원

주문 금액: 7,050,000원
7,050,000원 < 7,500,000원 → 주문 가능
```

### 증거금 홀딩

**API 호출:**
- `POST /api/v1/orders/buy` - 매수 주문

**처리:**
- 현금에서 2,820,000원 차감 (출금 가능 금액에서 제외)
- 잔여 출금 가능 현금: 180,000원
- 주문 상태: 대기

### 체결 (09:15)

**백그라운드 처리:** 체결 처리 로직 (내부)
- 삼성전자 가격 상승: 70,500원
- 체결 완료
- 보유 주식에 삼성전자 100주 추가
- 미결제 금액: 4,230,000원 (수요일 납부 마감 기한)

**현재 상태 확인:**
**API 호출:**
- `GET /api/v1/accounts/{accountId}` - 계좌 조회
- `GET /api/v1/accounts/{accountId}/holdings` - 보유 주식 조회
- `GET /api/v1/accounts/{accountId}/unpaid` - 미결제 내역 조회

**상태:**
```
총 현금: 3,000,000원
출금 가능 현금: 180,000원 (증거금 2,820,000원 홀딩 중)
보유 주식: 삼성전자 100주 (매수가 70,500원, 현재가 70,500원)
미결제: 4,230,000원
```

- 증거금 홀딩: 4,016,000원 (현금 180,000원 우선 사용 후 부족분은 매도 예정금액에서 담보 설정)

**현재 상태 확인:**
**API 호출:**
- `GET /api/v1/accounts/{accountId}` - 계좌 조회
- `GET /api/v1/accounts/{accountId}/unpaid` - 미결제 내역 조회
